Why CJM exists

Modern C++ is powerful. Everyday boilerplate should not be the cost.

Serialization, configuration, metadata, reflection-like workflows, and schema generation often force developers to write code that machines can generate reliably.

CJM removes that repetition without replacing C++, adding runtime reflection, or asking teams to adopt a new framework.

Architecture

Build-time generation, normal compiler workflow.

CJM sits inside the build. The parser, semantic analysis, metadata model, generator, and backend are implementation details.

What CJM does

CJM turns a small, explicit C++ input surface into generated serialization code that your compiler can see and check.

  • Parses ordinary C++ source
  • Builds a metadata model
  • Generates ordinary C++ serialization code
  • Integrates with CMake

v0.1.0

The first minimal working release includes an end-to-end pipeline from C++ input to generated JSON serialization code.

Example

struct User {
    std::string name; // json:"name"
    int age;          // json:"age"
};
add_executable(app main.cpp)

cjm_generate(
    TARGET app
    HEADERS user.hpp
)
nlohmann::json json = user;

User restored;
json.get_to(restored);

Generated to_json and from_json support normal nlohmann::json usage, including assignment and get_to.

Principles

Engineering before magic: CJM should feel native, predictable, and easy to inspect when the generated code matters.

Ordinary C++ should be enough

Users write standard C++. Generated output is standard C++. Everything in between is CJM's responsibility.

Build-time instead of runtime

No hidden runtime, no dynamic reflection engine, and no runtime cost introduced by CJM.

Visible and predictable

Generated code should be readable, inspectable, and debuggable when developers need to look inside.

Native to existing builds

Adding CJM should feel like enabling another compiler tool inside an existing CMake project.

Long-term vision

Code generation should feel like the build simply got smarter.

The first goal is build-time JSON code generation. The larger goal is a reusable foundation for modern build-time code generation in C++: simple user experience, production-quality engineering, and zero runtime overhead introduced by CJM.

CJM succeeds when developers forget that code generation is happening. They write standard C++, run their normal build, and everything works.

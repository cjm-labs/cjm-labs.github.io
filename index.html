CJM

Build-time JSON code generation for Modern C++.

Standard C++ in. Standard C++ out.

CJM lets you write ordinary C++ structs and generate ordinary C++ serialization code during the build.

GitHub v0.1.0 notes 
struct User {
    std::string name; // json:"name"
    int age;          // json:"age"
};

What CJM does

CJM turns a small, explicit C++ input surface into generated serialization code that your compiler can see and check.

  • Parses ordinary C++ source
  • Builds a metadata model
  • Generates ordinary C++ serialization code
  • Integrates with CMake

v0.1.0

The first minimal working release includes an end-to-end pipeline from C++ input to generated JSON serialization code.

Metadata model Code generator cjm generate cjm_generate(...) Golden tests Backend: nlohmann/json

Example

struct User {
    std::string name; // json:"name"
    int age;          // json:"age"
};
add_executable(app main.cpp)

cjm_generate(
    TARGET app
    HEADERS user.hpp
)

Generated to_json and from_json support normal nlohmann::json usage, including assignment and get_to.

Philosophy

CJM is architecture-driven: the parser, generator, and backend are implementation details. The user workflow should stay build-time, inspectable, and grounded in standard C++.
